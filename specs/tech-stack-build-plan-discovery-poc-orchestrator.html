Agent architecture — who does what

🎯 Orchestrator Agent Native · react style
Main entry point. Receives transcript, routes to sub-agents, collects results, presents readiness score.
↓ collaborates with
📋 Extraction Agent
Tools: extract_requirements, detect_gaps, flag_risks
→
📄 Artifact Agent
Tools: generate_placemat, generate_architecture, generate_checklist, draft_email
→
📊 Readiness Agent
Tools: calculate_readiness_score, provision_techzone (MCP)
Why 3 sub-agents? Each agent has a single responsibility. The Orchestrator only routes. This makes it easy to demo each piece independently and add more agents later (e.g. a Stakeholder Routing agent).

Key decisions made for you

Use WXOFile for transcript upload. The ADK natively supports file uploads to tools — no need to build a separate upload UI. The Extraction Agent's first tool accepts WXOFile and the webchat renders an upload button automatically.
Use react style agents. For orchestration flows that need to call tools in sequence and reason between steps, style: react is the right choice. The default style is better for simple Q&A.
TechZone MCP is already connected in this workspace. You don't need to build that integration — just add provision_techzone_request as a tool on the Readiness Agent and point it at the existing MCP server.
