Why stay on Streamlit (not wxO agents)

The existing app.py is already the right foundation. Porting to watsonx Orchestrate ADK adds a week of setup for the same demo outcome. Keep Streamlit for the PoC — it's fast, it's already wired to watsonx, and the UI is clean. Add wxO agents in Phase 2 as the "production path."
The TechZone MCP is in .bob/mcp.json — not Streamlit. To wire it from Streamlit, call the MCP directly via Python subprocess, or use Bob to expose a thin Python wrapper. Alternatively: demo the TechZone step separately via Bob chat, then splice it into the screen recording. Perfectly acceptable for a PoC.

Do not build: Custom frontend, Salesforce sync, Monday.com, RFP automation, vector search / RAG, custom auth. Each of these is a full day of debugging on its own. Mention them as "Phase 2" in the last slide of your demo deck.
